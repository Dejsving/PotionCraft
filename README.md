# PotionCraft
ПАСПОРТ ПРОЕКТА

Название: PotionCraft

Источник правил (Бизнес-логика) [1]: https://www.google.com/url?sa=E&q=https%3A%2F%2Fdnd.su%2Fhomebrew%2Farticles%2Fhomebrew%2F109-gerbalizm-i-alximiia%2F

Репозиторий[2]: https://github.com/Dejsving/PotionCraft

Технологический стек: C# (.NET), HTML, CSS, JavaScript (классическое веб-приложение с многослойной архитектурой).

1. ЦЕЛЬ ПРОЕКТА

Создание программного обеспечения (веб-инструмента) для автоматизации Homebrew-механик «Гербализма и Алхимии» для настольной ролевой игры Dungeons & Dragons 5-й редакции. Приложение предназначено для Мастеров Подземелий (DM) и игроков, чтобы снять рутину с бросков костей, подсчета модификаторов сложности, генерации лута (трав) в зависимости от экосистемы (биома) и расчетов при крафте зелий.

2. АРХИТЕКТУРА РЕПОЗИТОРИЯ (На основе текущей структуры)

Проект разделен на логические слои (N-Tier архитектура):

PotionCraft.Contracts — слой абстракций. Содержит интерфейсы, DTO (Data Transfer Objects) и доменные модели (Ингредиенты, Биомы, Зелья, Яды).

PotionCraft.InitData — слой инициализации. Отвечает за seeding (заполнение) базы данных или in-memory коллекций начальными данными. Содержит перенос всех таблиц из оригинальной статьи (список трав, их редкость, модификаторы сложности, таблицы экосистем).

PotionCraft.Repository — слой доступа к данным. Паттерн Репозиторий для получения информации об ингредиентах и таблицах лута.

PotionCraft — основной исполняемый проект (веб-интерфейс, контроллеры/сервисы, UI на HTML/CSS/JS). Содержит точки входа и логику взаимодействия с пользователем.

PotionCraft.Tests — слой модульного и интеграционного тестирования бизнес-логики.

3. ДОМЕННАЯ МОДЕЛЬ И БИЗНЕС-СУЩНОСТИ (Entities)

Для генерации задач аналитику необходимо оперировать следующими сущностями:

Ингредиент (Ingredient)

Атрибуты: Название, Редкость (Обычный, Необычный, Редкий, Очень редкий, Легендарный), Эффект (Описание), Тип (Эффект зелья, Модификатор зелья, Эффект яда, Модификатор токсина, Магический), Сложность (целое число со знаком, например, +2 или -1), Местность (Биом).

Экосистема / Биом (Ecosystem)

Виды: Арктика, Берег/Подводная среда, Пустыня, Лес, Луг, Холмы, Горы, Болото, Подземье, Обычные ингредиенты.

Логика: В каждом биоме есть таблица выпадения лута (бросок 2к6).

Зелье / Яд (Potion / Poison)

Состав (Обычное): 1 Базовый ингредиент (Эффект) + от 0 до 3 Ингредиентов-модификаторов.

Состав (Магическое): Элементальная вода (База) + 1 Магический ингредиент. (Модификаторы применять нельзя).

Атрибуты: Итоговая сложность (Алхимический опыт), Эффект, Цена.

4. КЛЮЧЕВАЯ БИЗНЕС-ЛОГИКА И АЛГОРИТМЫ (Для реализации)

Эпик 1: Гербализм (Сбор ингредиентов)

Алгоритм сбора: Выбирается биом -> Кидается 2к6 (имитация броска двух шестигранных кубиков, распределение от 2 до 12). Результат сопоставляется с таблицей биома.

Особое правило "Элементальная вода": Если при броске 2к6 на генерацию лута выпадает результат 2-4 или 10-12, система должна сделать дополнительный скрытый бросок 1к100 (d100). Если выпадает > 75, найденный ингредиент принудительно заменяется на «Элементальную воду»[1][2].

Дополнительные условия биомов (Парсинг правил из README):

Генератор должен учитывать текстовые модификаторы: «2 шт.», «1-2 шт.» (рандом), «Только на берегу», «Ночью 2 шт., днем перебросить», «2 шт. в пещерах», «2 шт. если дождь»[2]. Это требует передачи в метод сбора параметров окружения (время суток, погода, точная локация).

Количество добытого: При успешной проверке навыка (СЛ 15) добывается 1к4 порций сгенерированного по таблице ингредиента (базово)[1].

Эпик 2: Идентификация

Для каждого найденного предмета кидается проверка: СЛ = 10 + Сложность ингредиента[1]. Успех на +5 дает полные данные[1]. (Система должна уметь скрывать свойства предмета от игрока до успешной идентификации).

Эпик 3: Алхимия (Крафтинг)

Проверка Алхимического опыта (Сложность варки): СЛ крафта = 10 + [Сумма модификаторов сложности всех выбранных ингредиентов][1].

Валидация рецептов:

Обычное зелье/яд: Допускается строго 1 ингредиент с пометкой "Эффект" (База) и максимум 3 ингредиента с пометкой "Модификатор"[1].

Магическое зелье: Строго 1 "Элементальная вода" + 1 "Магический ингредиент"[1]. Блокировка добавления модификаторов (если попытаться — выдавать предупреждение о "Волне дикой магии" или порче зелья)[1].

Эпик 4: Экономика и Торговля

Продажа трав: Время поиска покупателя зависит от редкости (Обычный - 1к4 дней, Очень редкий - 1к12 дней). Итоговая цена вычисляется броском 1к100 с модификатором редкости[1].

Продажа зелий: Зависит от количества ингредиентов в зелье (1 ингр. = +10 к броску цены, 4 ингр. = -20 к броску)[1]. Итоговая цена — сумма стоимостей всех трав, умноженная на коэффициент сделки (от х0.1 до х2.0)[1].

5. ИНСТРУКЦИЯ ДЛЯ АНАЛИТИКА: КАК ГЕНЕРИРОВАТЬ ЗАДАЧИ

При формировании задач (User Stories, Tasks, Bugs) для разработчиков на основе этого описания, аналитик должен придерживаться следующих шагов:

Задачи для слоя InitData (Данные):

Создать задачи на перенос списков Ингредиентов Зелий, Ингредиентов Ядов и Магических Ингредиентов в C# классы (или JSON/БД сиды).

Создать задачи на маппинг таблиц распределения 2к6 для всех 10 экосистем[1][2]. Обязательно указать в задаче реализацию структуры (модели) для парсинга дополнительных условий ("Дождь", "Ночь" и т.д.)[1][2].

Задачи для слоя Contracts и Repository (Бэкенд):

Сформировать интерфейсы для IHerbalismService (генерация лута с учетом d100 на Элементальную воду[1][2]) и IAlchemyService (расчет сложности, валидация 1 база + до 3 модов[1]).

Реализовать "Кубик" (Dice Roller) — сервис для программной генерации случайных чисел (1к4, 2к6, 1к100) с кривой вероятности (2к6 имеет нормальное распределение, в отличие от 1к12)[2].

Задачи для слоя PotionCraft (Пользовательский интерфейс):

Разработать форму "Генератор трав": Dropdown с выбором биома, чекбоксы модификаторов среды (Идет дождь? Ночь? У воды?), кнопка "Сгенерировать лут".

Разработать форму "Алхимический стол": Drag & Drop или Multi-select интерфейс выбора трав. Автоматический подсчет итоговой СЛ рецепта в реальном времени. Валидация недопустимых комбинаций.

Задачи для слоя Tests:

Unit-тесты на проверку замены ингредиента на Элементальную воду (проверка срабатывания вероятности 25% при нужных бросках).

Unit-тесты калькулятора алхимии (например, 1 база + 4 мода должны вызывать ValidationException).

Данный паспорт проекта содержит исчерпывающий контекст бизнес-логики и архитектуры. Используйте его как Source of Truth при постановке задач в таск-трекер.
